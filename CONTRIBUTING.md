# Contributing to QuickCart

Thanks for your interest in contributing to QuickCart! We welcome improvements, bug fixes, documentation updates and tests.

Guidelines:
- Fork the repository and create a branch for your feature or fix: `git checkout -b feature/your-feature`
- Make small, focused commits with clear messages.
- Ensure your code follows the existing project style. Keep HTML/CSS simple and PHP procedural code consistent.
- Add documentation for any public-facing changes (update `README.md` or add docs).
- Open a Pull Request with a clear description of the change and any steps to test.

If your changes affect database schema or seed data, include SQL migration instructions or scripts.
